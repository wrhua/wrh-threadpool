1. FutureTask 实现了 RunnableFuture 接口，RunnableFuture接口继承于 Runnable、Future接口，

     

 

2. FutureTask实现返回值的原理：FutureTask 实现了Runnable接口，实际上就是在 Runnable#run 方法中调用 Callable#call 方法，得到

结果值后，将结果值保存到对象属性中，然后再通过 Future#get 返回执行结果；

 

3. FutureTask也有一个任务进度的状态值，其状态机如下：

     3.1 这是任务正常执行的情况，最终态是NORMAL；

             NEW -> COMPLETING -> NORMAL

     3.2 这是任务执行抛出异常的情况，终态是EXCEPTIONAL；

             NEW -> COMPLETING -> EXCEPTIONAL

     3.3 任务被取消；

             NEW -> CANCELLED

     3.4 任务被中断；

             NEW -> INTERRUPTING -> INTERRUPTED

3.5 任务的起始状态都是NEW：新建状态；

 

4. FutureTask#run方法：

public void run() {
    /*
        任务state不是新建状态
            或
        或设置运行线程失败，runner变量存储执行该任务的线程
     */
    if (state != NEW
            || !UNSAFE.compareAndSwapObject(this, runnerOffset, null, Thread.currentThread()))
        // 直接返回，不执行，因为状态不是新建，或者有运行线程，则证明任务已经被执行中，避免重复执行
        return;

    try {
        // 获取要执行的任务，构造时传入的callable
        Callable<V> c = callable;
        /*
            任务不为null
                且
            任务是新建状态
           double-chack
         */
        if (c != null && state == NEW) {
            V result;
            boolean ran;
            try {
                // 调用Callable#call方法，得到结果
                result = c.call();
                // ran = true，表示运行成功
                ran = true;
            } catch (Throwable ex) {
                // 运行失败
                result = null;
                // ran = false，表示任务执行失败
                ran = false;
                // 记录异常
                setException(ex);
            }
            // 如果运行成功
            if (ran)
                // 将结果保存起来，并唤醒在等待结果的线程
                set(result);
        }
    } finally {
        // runner置为null，即执行该任务的线程
        runner = null;
        // 判断任务是否是中断状态，是的话则尝试中断任务
        int s = state;
        if (s >= INTERRUPTING)
            handlePossibleCancellationInterrupt(s);
    }
}

 

5. 保存结果 set(result)：

protected void set(V v) {
    // 将状态从NEW设置到COMPLETING
    if (UNSAFE.compareAndSwapInt(this, stateOffset, NEW, COMPLETING)) {
        // 使用outcome成员变量保存任务执行结果
        outcome = v;
        // 状态从COMPLETING变成NORMAL，正常情况下的终态
        UNSAFE.putOrderedInt(this, stateOffset, NORMAL);
        // 唤醒阻塞在Future#get方法的线程
        finishCompletion();
    }
}

 

6. finishCompletion：唤醒阻塞在Future#get方法的线程，可以有多个线程调用Future#get，阻塞等待的线程会形成一个WaitNode链表；

private void finishCompletion() {
    // 获取正在等待的节点：即调用Future#get方法，正在阻塞中的线程
    for (WaitNode q; (q = waiters) != null;) {
       
        // 将waiters设置为null，表示已处理
        if (UNSAFE.compareAndSwapObject(this, waitersOffset, q, null)) {
            // 有可能有多个线程在等待结果，因此，waiters实际上是一个链表结构
            for (;;) {
                // 获取阻塞的线程
                Thread t = q.thread;
                if (t != null) {
                    // 唤醒线程
                    q.thread = null;
                    LockSupport.unpark(t);
                }
                // 获取下一个节点
                WaitNode next = q.next;
                if (next == null)
                    break;
                // 将链表断开，加快GC
                q.next = null;
                q = next;
            }
            break;
        }
    }

    done();

    callable = null;   
}

 

7. get方法：获取Callable执行结果；

public V get() throws InterruptedException, ExecutionException {
    int s = state;
    // 如果任务还没有执行完成，则阻塞，直到done，完成为止
    if (s <= COMPLETING)
        s = awaitDone(false, 0L);
    // 获取任务执行结果
    return report(s);
}

 

8. report：尝试获取执行结果；

private V report(int s) throws ExecutionException {
    // 从成员变量中获取执行结果
    Object x = outcome;
    // 如果任务执行成功，直接返回结果
    if (s == NORMAL)
        return (V)x;
    // 如果任务被取消，或是中断，抛出CancellationException异常
    if (s >= CANCELLED)
        throw new CancellationException();
    // 否则，说明任务有异常，将任务的异常 x 抛出
    throw new ExecutionException((Throwable)x);
}
